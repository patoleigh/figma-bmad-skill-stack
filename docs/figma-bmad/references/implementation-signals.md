# Implementation Signals

Look for visible signs that the design is implementable enough.

## Positive signals

- sections are clearly grouped
- the primary interaction is visually evident
- reused components and patterns are consistent
- naming is clear enough to map intent
- state behavior is visible where needed
- spacing and layout are consistent with the file

## Weak signals

- polished visuals with unclear behavior
- one-off local exceptions
- component reuse that hides role mismatch
- unresolved canonical source conflict

## Working rule

Readiness should follow visible signals in the file, not assumed knowledge outside it.
