# Screen Structure

## Core rule

- Build the screen structure before choosing detailed variants or styling.

## What to define first

- main sections
- primary interaction zone
- supporting content areas
- feedback or status areas
- secondary controls only if they support the main task

## Structure checks

- Is the main action obvious?
- Does the hierarchy support the intended flow?
- Are related elements grouped by function?
- Does the screen match the structure of similar screens already in the file?

## State coverage

At minimum, review whether the screen needs:

- default
- loading
- empty
- error
- success
- disabled

If one of these matters to the scoped interaction and is missing, the composition is incomplete.
