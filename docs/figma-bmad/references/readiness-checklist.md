# Readiness Checklist

Use this as a compact gate, not as a broad audit.

## Core checks

- scope is explicit
- structure is clear
- main interaction is obvious
- reused system assets are identifiable
- naming is clear enough for handoff
- key states are present where needed
- no critical contradiction is visible

## Status guide

- `ready`: implementation can proceed without critical guessing
- `partial`: mostly clear, but some clarification is still needed
- `blocked`: implementation would need to invent or choose between conflicting answers
